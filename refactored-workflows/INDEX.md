# 重构工作流索引

> 原始工作流 → 精简优化 → 记录改了什么、为什么

## 重构原则

1. **最少节点** — 能少一个节点就少一个
2. **清晰数据流** — 连线不交叉，从左到右
3. **可复用模块** — 常用组合抽成 Group Node
4. **参数外露** — 常调参数用 Primitive Node 外露
5. **注释完整** — 每个关键节点写 Note 说明

## 重构清单模板

```markdown
## 重构：XXX 工作流
- **原始**: 来源 + 节点数
- **重构后**: 节点数 + 主要变化
- **改动清单**:
  1. 删除了 XX（因为 YY）
  2. 替换了 XX 为 YY（因为 ZZ）
  3. 合并了 XX + YY（功能重复）
- **效果对比**: 速度/质量/可维护性
- **设计原理**: 为什么这样连
```
